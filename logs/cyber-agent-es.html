Cyber Agent ES

*Q1.本インターンにご応募いただいた理由を教えてください

一言で言えば挑戦です｡
自分の実力がどこまで通じるのか知りたい｡同世代からは得られない経験則からのアドバイスや慣れ親しんだ友人に囲まれた今の環境とは明確に異なる世界に飛び込んでみたい｡それができるのが業務レベルの体験ができるのが就業型インターンだけだと考えたからです｡その中でもCyberAgentさんのインターンに応募した理由は､1ヶ月という長い期間で実際の業務に携われるという環境に惹かれた点と､CyberAgentが持つ若手や新卒にもチャンスを与え､挑戦できる職場である点が私の中で魅力的に映ったからです｡

私の高校時代は挑戦とは真反対の内向型の人間でした｡成果物を外に発表することなく､ネットでエンジニアが集まるDiscordサーバーなどに参加することもなく､孤独にプログラミングをしていました｡しかし大学に入って同級生に技術力もアクティブさも私を超える友人に出会いました｡そのとき私は悔しい感情よりも追いつこうというやる気に満ち溢れた気分になりました｡それから私はサークル内でアクティブに活動したり､サークル外のエンジニアの集まりに参加したりと積極的に活動しています｡そして次は社会人との交流したいという挑戦の形として本インターンに応募しました｡

※ フォームの方に2028年卒がなかったので2027年卒とプルダウンを選択しました｡学部1年です｡

*Q2.本インターンで経験したいこと、挑戦したいことを教えてください。

「特定のサービス、技術を経験したい」、「チーム開発や設計などに挑戦したい」などでもけっこうです。また複数記載いいただいても結構です。

Q1でも話した通り実務経験を積みたいと私は考えています｡細かなエラー修正から新機能の追加､設計､移行計画など幅広く実務で行う作業に興味があります｡その中でも一番挑戦したいことは大規模な会社でしか出来ないようなプロダクトを生み出す､改善に携わることです｡なぜなら個人開発では作ることができないような巨大プロダクトの制作に携われて､そこでしか味わえない経験や知見を広げられると考えているからです｡

そしてもう一つが､設計や計画の会議に参加することです｡私は意見を他の人に理解できるように伝えることに苦手意識を持っていました｡ですが最近サークルの総会にて私が立ち上げた競プロ部というプロジェクトについて全員の前で発表をしました｡緊張しましたが伝えなければいけないことを伝えることが出来たと振り返って思います｡この経験から私は自分の意見をチームメンバーに伝えるということの重要性とその難しさを実感しました｡だからこそ私は会議などの話し合いで意見の言える人間になりたいと思いCyberAgentさんのインターンではぜひ挑戦したいと考えています｡

fookeys

[共同開発][一年間][プログラム全て][Webで動作する1対1の対戦カードゲーム][Vue + firebase]
この開発は高校の課題研究で取り組みました｡4人チームでしたが内容は4人で考え､絵､音楽を除く全てを私が担当しました｡Webフロントエンドという点では個人開発に近いと思ったので書かせてもらいました｡fookeys - github

内容について

ゲームのコンセプトとして､ ｢フードロス｣ × ｢ゲーム｣ を組み合わせることでフードロスを小中高生にも身近に感じてもらい､理解を深めて貰うのがテーマです｡そのため学校配布のiPadやタブレットPCでも動作するWebブラウザという仕様要件にしました｡コンセプトをゲームのシステムに落とし込み､ゲームで勝利するためにはフードロスを減らす行動をプレイヤーが自主的に選択してもらうために､様々な工夫を凝らしました｡

カード

カード一枚一枚を食べ物に見立て､このゲーム特有の概念として､腐り値と呼ばれる値が存在し､カードの左上に表示されています｡腐り値とは､ドローした時点で3~5程度の数値で､ターンが経過する毎に数値が1ずつ減り､0になると腐ったカードに変化し､使用不可能のカードとして手札に残ります｡このカードゲームは手札の最大上限が9枚までとなっていて､手札を圧迫してしまいます｡一般的にコストと呼ばれるものを満腹値として表現しています｡

買い物フェーズ

両プレイヤーのターンが終了すると､買い物フェーズに移行します｡買い物フェーズではランダムに表示される3枚のカードの中から0~3枚のカードを手札に加える事ができます｡この仕様が､先ほど紹介した腐り値と相乗効果を発揮しこのゲームをより一層おもしろいものにします｡買い物フェーズでたくさん買いすぎると満腹値の上限に達してしまい食べきることができず､余ったカードが腐ったカードに変化しゲームが不利になってしまいます｡

ミッション

ゲーム開始時から画面右下に3つ表示されています｡両プレイヤーとも同じミッションが表示されており､達成度の進捗も両プレイヤーで共通です｡ミッションをクリアするとギフトの使用に必要な貢献度が手に入ります｡

ギフト

ギフトとは､ゲームの選択肢を広げ駆け引きにエッセンスを加える役割があります｡ギフトを使うために必要な貢献度は基本的にミッションをクリアすることで手に入ります｡つまりミッションをクリアした方がゲームを有利に進められる仕組みになっています｡

まとめ

このようにゲームのシステムにフードロスを落とし込んで､ゲームで勝利するにはフードロスを減らす行動をプレイヤーが自主的に選択する様々な工夫を凝らしました｡SDGs探究AWARDS2023に応募して､このゲームは審査員特別賞を頂きました｡

役割について

ゲームシステムやコンセプトはチームメンバー全員で考え､絵や音楽は他の人にやってもらいました｡私はコーディングを全て担当しておりフロントエンドにVue3.3とBaaSにfirebaseを用いて制作しました｡

*Q8.自信のある技術領域があればその技術領域における経験についてご記載ください。

私が得意としている技術領域はReactです｡先日友人と10日間のハッカソンを行いました｡Reactでは関数コンポーネントの作成方法がいくつかあったり､ファイルごとに一つのコンポーネントという設計思想があるなど､人によって違いがあります｡そこで私はこのプロジェクトではこの書き方をするというESLintとPrettierやドキュメントを作成して全体で統一しました｡

*Q10.最近興味を持っている技術について、興味を持っている理由と、その技術において新しくチャレンジしていることや学習していることがあれば教えてください。(分野/粒度不問)

私は今興味を持っているものはSSGなどの静的レンダリングです｡私はブログサイトの技術選定をしていました｡SSGがしたいと思いNextjsを選択しようと考えたのですが､SSGを行うためにNextを使うのは少々ハイスペックすぎる気がしました｡まずNextjsの良いところはSSGができる他に､Vercelとの連携やImageの最適化があると思います｡ですがHMRが遅すぎてDXが悪く他のフレームワークを探しました｡そして見つけたのが､Wakuとministaです｡

WakuというのはjotaiというReact状態管理ライブラリの制作者が制作している日本製のReactフレームワークです｡SSGなどの基本機能のみをサポートしているので､Wakuを使用すると軽量さとSSGを両立することができるのです｡他にもSSRにも対応しているのでレンダリングのみを求めているのであれば今後Wakuで事足りる場合が多くなると考えています｡Wakuは現在Beta版ですがそれを上回るほどに私は魅力を感じています｡

ministaはqracさんが制作しているjsxからHTMLを作るスタティックサイトジェネレーターで､ビルドすると静的なHTMLに変換して一部ファイルのインポート方法を変更するだけで一部分をReactコンポーネントとして認識することができます｡HeadlessCSSやMantineはビルド時に全てを出力するので相性が良くて採用しました｡

フロントエンドは決まったのですが,ブログのCMSにObsidianを採用しようと思い現在試行錯誤中です｡
